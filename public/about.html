How it works

c3-ssg treats a single folder as your whole project. Put HTML, CSS, JavaScript, images, and fonts in public/ and that is your site.

Project layout

public/        ← your site (edit this)
  index.html
  about.html
  styles.css
  app.js
  assets/
dist/          ← built output (generated)
server.mjs     ← production static server
scripts/build.mjs

Local development

npm run dev serves public/ directly with live-reload, so the browser refreshes as you edit. No build step needed while developing.

Production build

npm run build copies public/ into dist/, precompresses text assets to Brotli and gzip, and adds the files a host needs. npm run start (or npm run preview) serves that output exactly as production would.

Deployment

Push to GitHub and the included workflow builds and publishes to GitHub Pages automatically. Netlify, Vercel, and Docker configurations are included too — pick whichever free host you prefer.
